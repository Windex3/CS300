# CS300
### What was the problem you were solving in the projects for this course?
The primary problem that I was solving in these projects was managing and organizing computer science course data efficiently. Project one focused on analyzing different data structures to determine which structure was the best for storing and retrieving course information. Project two was implementing a working program that loads course data, stores it in a binary search tree, and prints the courses in alphanumeric order. 

### How did you approach the problem? Consider why data structures are important to understand.
I approached the problem by first reviewing the strengths and weaknesses of vectors, hash tables, and binary search trees. Understanding data structures is important because they directly impact program efficiency, scalability, and memory usage. In the end, I chose the binary search tree mainly because it offers native support for in order traversal, which was a main requirement of the program. 

### How did you overcome any roadblocks you encountered while going through the activities or project?
One of the main challenges was ensuring the CSV file loaded correctly and that the tree operations worked without causing infinite loops or invalid memory access. I addressed these issues by adding input validation and carefully tracing the tree traversal logic.

### How has your work on this project expanded your approach to designing software and developing programs?
This project reinforced the importance of planning before coding. Writing pseudocode for the binary search tree before beginning to code it made implementing it a lot easier. 

### How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
Through this work, I placed greater emphasis on clear function structure, meaningful variable names, and modular design. Separating responsibilities like file loading, searching, and printing made the final program a lot easier to read.
